# 个人主页前后端分离系统使用说明

## 项目简介

本项目是一个基于HTML/CSS/JavaScript开发的个人主页系统，实现了前端展示与后端管理完全分离的架构。前端展示页简洁干净，只展示内容；后端管理页提供完整的内容编辑功能，包括文本编辑、图片上传等。

## 项目结构

```
个人主页展示（二）/
├── index.html              # 前端封面页
├── readme.md              # 项目说明文档
├── 使用说明.md           # 使用说明文档
├── css/                   # 样式文件目录
│   └── style.css          # 主样式文件
├── html/                  # 前端页面文件目录
│   ├── home.html          # 个人主页主页面
│   ├── hobbies.html       # 兴趣爱好页面
│   ├── message_room.html  # 留言书屋页面
│   └── learning_log.html  # 学习日志页面
├── img/                   # 图片资源目录
└── admin/                 # 后端管理页面目录
    ├── login.html         # 管理登录页面
    └── index.html         # 管理主页面
```

## 功能说明

### 1. 前端展示页

- **封面页**：展示个人姓名和学校信息，提供进入主页的引导按钮
- **个人主页**：显示个人介绍和联系信息
- **兴趣爱好**：展示兴趣爱好内容，包括电影、阅读、摄影、运动和心动时刻
- **留言书屋**：提供留言功能和日历展示
- **学习日志**：展示学习记录

### 2. 后端管理页

- **登录功能**：使用用户名和密码登录管理系统
- **个人信息管理**：编辑个人姓名、学校、专业、联系方式和个人介绍
- **兴趣爱好管理**：添加、编辑、删除兴趣爱好内容
- **电影管理**：添加、编辑、删除喜欢的电影
- **图片上传**：支持从电脑本地上传图片

## 部署方法

### 1. 本地部署

1. 将项目文件夹复制到本地任意位置
2. 使用浏览器打开 `index.html` 文件即可访问前端展示页
3. 使用浏览器打开 `admin/login.html` 文件即可访问后端管理页

### 2. 服务器部署

1. 将项目文件夹上传到服务器的网站根目录
2. 通过服务器域名或IP地址访问前端展示页（如：http://example.com/index.html）
3. 通过服务器域名或IP地址访问后端管理页（如：http://example.com/admin/login.html）

## 使用方法

### 1. 登录管理系统

1. 打开管理登录页面：`admin/login.html`
2. 输入用户名：`admin`
3. 输入密码：`123456`
4. 点击「登录」按钮进入管理主页面

### 2. 管理个人信息

1. 在管理主页面点击「个人信息管理」
2. 修改表单中的内容（姓名、学校、专业、联系方式、个人介绍）
3. 点击「保存个人信息」按钮
4. 刷新前端展示页，即可看到更新后的内容

### 3. 管理兴趣爱好

1. 在管理主页面点击「兴趣爱好管理」
2. 填写兴趣爱好名称、描述
3. 点击「点击上传兴趣爱好图片」上传图片
4. 点击「添加兴趣爱好」按钮
5. 兴趣爱好会显示在下方的列表中
6. 可以点击「编辑」或「删除」按钮对兴趣爱好进行管理

### 4. 管理电影

1. 在管理主页面点击「电影管理」
2. 填写电影名称、描述
3. 点击「点击上传电影封面」上传电影封面图片
4. 点击「添加电影」按钮
5. 电影会显示在下方的列表中
6. 可以点击「编辑」或「删除」按钮对电影进行管理

## 技术说明

1. **前端技术**：HTML5 + CSS3 + JavaScript
2. **后端技术**：使用浏览器的localStorage实现数据存储
3. **数据持久化**：所有数据保存在浏览器的localStorage中
4. **响应式设计**：适配不同屏幕尺寸
5. **纯静态实现**：无需服务器端支持，直接部署即可使用

## 注意事项

1. **数据存储**：所有数据保存在浏览器的localStorage中，如果清除浏览器缓存，数据会丢失。建议定期备份数据。
2. **图片大小**：上传的图片建议不超过2MB，过大的图片会影响页面加载速度。
3. **浏览器兼容性**：建议使用现代浏览器（Chrome、Firefox、Edge等）访问本系统。
4. **安全性**：本系统使用简单的用户名密码登录，仅适用于本地或内部使用，不建议在公共网络中使用复杂或重要的数据。

## 常见问题

### 1. 忘记管理密码怎么办？

答：管理密码保存在 `admin/login.html` 文件中，可以直接修改该文件中的密码。

### 2. 如何备份数据？

答：可以通过浏览器的开发者工具（F12），在Application或Storage选项卡中找到localStorage，将数据导出保存。

### 3. 如何恢复数据？

答：在浏览器的开发者工具中，将备份的数据导入到localStorage中。

### 4. 上传的图片不显示怎么办？

答：请确保上传的图片格式为JPG、PNG等常见图片格式，且大小不超过2MB。
